A Capacitor-powered Android app sharing 100% of the domain, use cases and design system with the desktop and web builds. Adds a **native foreground service** for background audio, a **MediaSession-aware notification**, lock-screen controls, **Bluetooth headset button support** and WASM-based song recognition.

## ✨ Highlights

- 📲 **Native shell, TypeScript core.** Capacitor 6 · Android SDK 34 · single TypeScript codebase.
- 🎵 **True background audio.** A `START_STICKY` foreground service keeps the stream alive when the screen locks or the app is backgrounded.
- 🔔 **MediaSession notification.** Lock-screen controls, headset buttons, and a persistent, non-dismissable playback notification — with cover art, station name and play/pause/stop actions.
- 🧩 **Two-layer media integration.** A custom `AudioPlayerPlugin` for the foreground service + `capacitor-music-controls-plugin-v3` for richer Bluetooth/MediaSession bindings. Graceful fallback if either layer fails.
- 🔋 **Battery-aware.** Visualizer RAF loops stop when the app goes to the background; audio continues through the native service.
- 🌐 **CORS-free streaming.** A native HTTP plugin (`NativeHttp`) bypasses WebView CORS for the Shazam signature POST.
- 🪶 **Bottom-nav layout.** Mobile-first shell with a bottom navigation bar, mini-player and expandable full-screen player sheet.
- ⬇️ **Download APKs from Releases.** Tag-driven GitHub Actions builds + signs a release APK.

## 📦 Download

Prebuilt signed APKs are attached to every tagged release on the [GitHub Releases](https://github.com/MbarkT3STO/Aether-Radio/releases) page.

- Minimum Android version: **Android 7.0 (API 24)**

## 🚀 Quick start

> **Prerequisites**: Node.js 20+, Java 21 (Temurin), Android Studio with SDK 34 + build-tools.

```bash
# Install dependencies
npm install

# Dev server (Vite, WebView loads it via Capacitor live-reload)
npm run dev

# Build web assets
npm run build

# Sync the web build into android/app/src/main/assets/public
npm run cap:sync

# Open the Android Studio project
npm run cap:open

# Or run straight onto a connected device / emulator
npm run cap:run
```

Inside Android Studio: pick a device, press ▶. Logcat filter `chromium` surfaces console output; `AudioForegroundService` and `AudioPlayerPlugin` tags surface native service logs.

### Capacitor config

```ts
// capacitor.config.ts
{
  appId: 'com.aetherradio.app',
  appName: 'Aether Radio',
  webDir: 'dist',
  server: { androidScheme: 'https', allowNavigation: ['*'], cleartext: true },
  android: {
    allowMixedContent: true,
    captureInput: true,
    webContentsDebuggingEnabled: false,
    backgroundColor: '#000000',
  },
}
```

`allowNavigation: ['*']` + `cleartext: true` let the WebView fetch Icecast/Shoutcast streams on both HTTP and HTTPS — this is the one area where Android is *more* permissive than Desktop/Web by necessity (Capacitor isolates the WebView from the platform network stack differently than Electron).

## 🎛️ Background audio — the two-layer design

Background audio on Android requires a proper **foreground service** with a persistent media-style notification. We implement this in two independent layers so a failure in either doesn't silence playback:

### Layer 1 — `AudioPlayerPlugin` (custom native plugin)

A Kotlin plugin we ship inside `android/app/src/main/java/…`. Exposes three methods:

```kotlin
startForeground(track, artist, cover)  // promotes the process to foreground + posts notification
pauseForeground()                      // updates notification state but keeps service alive
stopForeground()                       // tears down service + clears notification
```

Triggered from `MediaNotificationService.ts` on `player:play` / `player:pause` / `player:stop` EventBus events. Uses `androidx.media` for the `MediaSessionCompat` and a styled `MediaStyle` notification.

Notification button taps dispatch a JS custom event (`audioPlayerAction`) through the WebView so the TypeScript layer can update `PlayerStore`.

### Layer 2 — `capacitor-music-controls-plugin-v3`

Adds richer MediaSession integration: Bluetooth headset buttons, car HU controls, Android Auto, and additional system events. Loaded lazily — if it fails to import or initialize, the app keeps working through Layer 1.

### Background lifecycle

- On `appStateChange → inactive` we **stop every canvas RAF loop** (mini-player bar + expanded sheet visualizer). Audio keeps flowing through Layer 1.
- On `appStateChange → active` we restart the visualizer with a 200 ms settle delay.
- **AudioContext is never suspended** — suspending it would kill the stream. Stopping the RAF loop is enough to eliminate the Web Audio CPU cost.
- `CapApp.backButton` closes the expanded player sheet first, then navigates back, then minimizes the app — matching Android platform expectations.

## 🎙️ Song recognition on Android

Direct browser calls to Shazam fail CORS inside the WebView. Workaround:

1. PCM is captured from the `AnalyserNode` (like every other platform).
2. `shazamio-core/web` generates a Shazam signature locally (WASM).
3. The signature is POSTed through the native `NativeHttp` plugin (pure Kotlin `HttpURLConnection`) — no CORS, no preflight.
4. Results are parsed and rendered in a modal with deep-links to Apple Music, Spotify, Deezer, YouTube, YouTube Music and Shazam.

```
┌─── WebView ─────────────────────────────────────┐
│ AnalyserNode → shazamio-core WASM → signature    │
│                            │                     │
│                            ▼                     │
│                 NativeHttp.post(…) ──────────────┼────→ amp.shazam.com
│                            │                     │
│                            ▼                     │
│                    Parsed result → Modal         │
└──────────────────────────────────────────────────┘
```

## 🎨 Mobile-tuned design

Same tokens as Desktop/Web, with a bottom-nav-first layout and iOS-inspired bottom sheets:

- `BottomNav` — 5 tabs (Home · Explore · Search · Favorites · Settings), blur-glass background, accent-tinted active state.
- `MiniPlayer` — persistent 64dp bar just above the nav; tapping expands into a full-screen, gesture-dismissable sheet with album art, big transport controls, volume slider, sleep timer and song recognition.
- `StatusBar` — style and background color auto-adjust to the current theme (`Style.Dark` + `#000000` in dark mode, `Style.Light` + `#f2f2f7` in light mode).
- `data-accent` is restored from `localStorage` on first paint so the app tints correctly before the settings roundtrip completes.

## 🗄️ Persistence

All local state uses the `@capacitor/preferences` plugin (AndroidX `EncryptedSharedPreferences`):

- `favorites`, `history`, `settings`, `customStations` — identical shapes to Desktop/Web.
- `accent-color` — duplicated into `localStorage` for instant restore before settings hydrate.

## 🌐 Network & streaming

- `RadioBrowserApiClient` talks to a single configured mirror (default: `de1.api.radio-browser.info`) through `MobileStationRepository`.
- Stream playback goes through the WebView's `<audio>` element — Android's `allowMixedContent` lets HTTP streams play inside an HTTPS-scheme app.
- Song recognition traffic is routed through `NativeHttp` to bypass WebView CORS.

## 🏗️ Release builds & signing

Signing details live in [`RELEASE.md`](./RELEASE.md). Short version:

```bash
# 1. Generate a keystore once
keytool -genkey -v \
  -keystore android/aether-radio.keystore \
  -alias aether-radio \
  -keyalg RSA -keysize 2048 \
  -validity 10000

# 2. Add these GitHub Secrets
#    KEYSTORE_BASE64 · KEYSTORE_PASSWORD · KEY_ALIAS · KEY_PASSWORD

# 3. Tag and push — CI builds + signs + attaches the APK to a Release
git tag android-v1.1.0
git push origin android-v1.1.0
```

### Local signed build

```bash
cp android/keystore.properties.example android/keystore.properties
# ...edit keystore.properties with your passwords + alias...
cd android && ./gradlew assembleRelease
# → android/app/build/outputs/apk/release/app-release.apk
```

The Gradle signing config prefers env vars (`KEYSTORE_PATH`, `KEYSTORE_PASSWORD`, `KEY_ALIAS`, `KEY_PASSWORD`) for CI and falls back to `keystore.properties` locally.

| Permission | Why |
|---|---|
